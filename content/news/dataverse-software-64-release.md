The Dataverse Project is pleased to announce the release of version 6.4, continuing the momentum of the 6.x series with important bug fixes and new features that enhance the platform's functionality and reliability. This release demonstrates the project's ongoing commitment to responsive development that addresses community needs while maintaining the stability required for production research data repositories.

Version 6.4 introduces several valuable features that improve the user experience and expand administrative capabilities. Enhancements to data upload and download workflows make it easier for researchers to deposit and retrieve their datasets, while improvements to metadata handling support more comprehensive data documentation. New administrative tools provide repository managers with better oversight and control, and enhanced monitoring capabilities help identify and resolve issues proactively. The release also includes updates to external tool integrations, enabling researchers to connect their data with analysis platforms and visualization tools.

Bug fixes in this release address issues identified through community feedback and testing, improving system stability and resolving problems that affected specific configurations or workflows. Performance improvements ensure that repositories continue to operate efficiently under varying loads. Repository administrators are encouraged to review the release notes and plan their upgrades to benefit from these enhancements. The 6.4 release reinforces Dataverse's position as the trusted, open-source solution for research data management adopted by institutions worldwide.
